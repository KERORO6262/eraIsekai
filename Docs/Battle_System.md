# Battle_System.md
> 系統：抽牌組合式回合戰鬥  
> 核心：抽牌 4 格，組合映射技能，無技能時普攻，暫存槽每回合一次，自動戰鬥可切換策略  
> 進出：由日間或晚間 `CALL @BATTLE_ENTER` 進入，戰鬥結束 `RETURN` 回呼叫點

本文件定義戰鬥階段的「回合循環」與「決策邏輯」。戰鬥以固定 4 張手牌作為每回合操作面，玩家透過卡牌組合觸發映射技能，若沒有合適組合則會自動執行普攻，確保流程不會卡住。
暫存槽提供每回合一次的存取操作，用來把一張牌延後到下一回合使用，讓玩家能對抗壞手牌。自動戰鬥以策略權重驅動評分器，策略切換只影響選牌與技能偏好，不破壞底層規則。

---

## 1. 名詞與區塊
- **Deck（牌組）**：未抽出的牌堆。
- **Discard（棄牌堆）**：已用或回合結束丟棄的牌。
- **Hand Slots（手牌槽）**：固定 4 格，每回合抽滿。
- **Temp Slot（暫存槽）**：固定 1 格，可存 1 張牌跨回合保留。
- **Mapping（映射表）**：卡牌組合配方表，輸入卡牌集合，輸出技能 ID 與效果。
- **Basic Attack（普攻）**：回合保底行動，低命中或低傷害，避免無牌可打。

## 2. 回合流程（Turn Loop）
### 2.1 回合開始：抽牌 4 格
1. 清空手牌槽（上一回合未保留到 Temp 的牌會進棄牌堆）。
2. 從 Deck 依序抽牌，填滿 4 格手牌槽。
3. 若 Deck 不足以抽滿：
   - 將 Discard 洗回 Deck（重新洗牌）。
   - 繼續抽到滿 4 張。
4. 若 Temp Slot 內已有牌，仍然保留，不會自動塞進手牌。

> 設計意圖：每回合操作面固定，玩家的「策略」來自選牌與暫存，而不是手牌上限無限膨脹。

### 2.2 暫存槽（Temp Slot）：每回合一次存取（第三階段實作）
在戰鬥 UI 中，玩家可先透過 410~413 勾選手牌槽位（H0~H3），再按 `[450]` 執行暫存槽存/取。

- 每回合僅可成功操作一次：以 `TFLAG:TEMP_USED_THIS_TURN` 控制。
- 先檢查手牌選取數：**必須精確選取 1 個非空手牌槽位**，否則拒絕操作並提示。
- 若 Temp 為空：將該選取手牌元素移入 Temp，原手牌清空。
- 若 Temp 非空：執行交換，Temp 元素與該手牌元素互換。
- 成功後清空 `UI_HAND0_SEL~UI_HAND3_SEL` 與 `UI_TEMP_SEL`，並把 `TFLAG:TEMP_USED_THIS_TURN` 設為 1。

> 設計意圖：Temp Slot 不是第 5 張手牌，而是回合內一次性的節奏調整器；要求精確單選可避免多選造成的歧義與濫用。

### 2.3 技能映射：從 4 張牌中組合出技能
玩家在手牌槽中選擇 2 到 4 張牌，形成一個「組合集合」，並依下列規則判定技能：

1. **先比對精確配方（Exact Recipe）**  
   - 例如：`A + B`，或 `A + A + D`，或 `A + B + C + D`
2. **再比對類別配方（Tag Recipe）**  
   - 例如：任意兩張「攻擊類」觸發「連擊」
3. **若仍無命中**：本回合視為「無技能」，進入普攻流程

> 組合是「集合」概念，預設不看順序。

#### 2.3.1 映射表格式（建議）
- Key：卡牌 ID 或卡牌 Tag 的集合（排序後字串化）
- Value：技能 ID，消耗規則，效果參數

#### 2.3.2 映射技能例表（示意）
| 配方 | 技能名（示意） | 主要效果 | 額外規則 |
|---|---|---|---|
| A + B | 強力打擊 | 高傷害，低命中補正 | 用掉 A 與 B |
| A + A | 連斬 | 中傷害 2 段 | 若第一段命中才觸發第二段 |
| D + D | 格擋架勢 | 本回合獲得護盾 | 下回合第一擊減傷 |
| A + D | 反擊 | 低傷害，獲得反擊狀態 | 受攻擊時追加傷害 |
| S + A | 破甲 | 中傷害，附帶破甲 | 破甲持續 2 回合 |
| P + P | 專注 | 回資源或抽牌加成 | 下回合抽牌前生效 |

> A=攻擊牌，B=重擊牌，D=防禦牌，S=技巧牌，P=支援牌。日後再改用元素、武器類型或姿態類型。

### 2.4 未命中普攻（Basic Attack when no skill）
當玩家沒有組合出技能，或策略判定「不值得出技能」時，自動執行普攻，規格建議如下：

- **命中率**：偏低或不吃太多加成，讓普攻是保底，不是主力
- **傷害**：偏低，但仍能推進戰鬥節奏
- **可受狀態影響**：例如破甲、虛弱仍會影響普攻，保持系統一致性
- **必要時加速戰鬥**：若連續多回合普攻，給一個隱性補正，避免對局無限拖

> 這個機制的目的不是懲罰玩家，而是保證回合一定能結算完，讓戰鬥模組永遠可回 `RETURN`。

### 2.5 回合結束：棄牌與狀態更新
- 被消耗的組合牌進 Discard。
- 未使用的手牌建議也進 Discard（簡化且節奏明快）。  
  - 例外：Temp Slot 內的牌保留，不進 Discard。
- 更新 Buff/Debuff 回合數。
- 進入敵方回合或下一回合。

## 3. 暫存槽的「一次存取」細節（可直接照規則做 UI）
### 3.1 操作窗口設計
- 在每回合抽牌後，顯示：
  - Temp Slot 目前的牌（若有）
  - 4 格手牌
  - 兩個選項：存牌，取牌
- 玩家成功執行一次後，該窗口鎖定到回合結束，避免二次操作。

### 3.2 常見邊界情況
- Temp 空，玩家選取牌存入 Temp：手牌槽空 1 格，仍可用 3 張組合技能。
- Temp 有牌，玩家取回到滿手牌：採交換，確保總牌量不超出「手牌 4 + Temp 1」的上限。
- 玩家不操作 Temp：視為放棄本回合 Temp 行動，直接進入組合階段。

## 4. 自動戰鬥策略（Auto Battle）
自動戰鬥的核心不是「寫死技能優先序」，而是「列舉本回合所有可用行動並打分」。策略只改變權重，這樣要加新牌與新技能時，不必重寫 AI。

### 4.1 策略清單
1. **進攻優先（Aggressive）**
   - 偏好高傷害映射技能
   - 可接受較高資源消耗
   - 低血仍可能硬打，除非低於危險線

2. **防禦優先（Defensive）**
   - 偏好護盾、減傷、控制類技能
   - 會提高生存權重，降低冒險行動
   - 血量越低越保守

3. **節省資源（Conservative）**
   - 優先低消耗或回資源技能
   - 只有在能確定擊殺或優勢很大時才爆發
   - 會更常使用 Temp Slot 保存關鍵牌

### 4.2 行動列舉（Move Generation）
每回合生成候選行動集合，至少包含：
- 不用 Temp，直接用手牌任意 2 到 4 張組合出的所有技能
- 若 Temp 可用，加入以下分支之一：
  - 先存一張牌，再嘗試用剩下牌組合技能
  - 先取一張牌（交換），再嘗試用新手牌組合技能
- 若沒有任何技能候選：加入「普攻」

### 4.3 評分器（Scoring）
每個候選行動算一個分數，策略用不同權重：

- 進攻得分：`DamageScore`
- 防禦得分：`GuardScore`
- 控制得分：`ControlScore`
- 資源得分：`ResourceScore`（回資源加分，耗資源扣分）
- 風險扣分：`RiskPenalty`（低命中，容易反噬，或會導致下回合手牌更差）

總分示例：
- **進攻優先**：`2.0*Damage + 0.6*Guard + 0.4*Control + 0.3*Resource - 1.0*Risk`
- **防禦優先**：`0.9*Damage + 2.0*Guard + 0.8*Control + 0.4*Resource - 0.6*Risk`
- **節省資源**：`1.1*Damage + 0.9*Guard + 0.6*Control + 1.8*Resource - 0.8*Risk`

> 權重只是建議，真正的好手感來自技能數值與資源曲線的協調。

### 4.4 策略切換規則（Algorithm）
策略可由玩家手動切換，以「手動優先，自動補位」：

1. 若玩家本場設定「鎖定策略」：永遠用該策略。
2. 否則進入自動切換：
   - 若我方 HP 比例 <= 30%：切到防禦優先
   - 若我方資源比例 <= 20%：切到節省資源
   - 若敵方 HP 比例 <= 25%：切到進攻優先（收頭）
   - 其餘：維持上一回合策略或均衡策略

### 4.5 Temp Slot 在 AI 中的使用偏好
- 進攻優先：Temp 主要用來「保留爆發關鍵牌」，例如重擊或破甲。
- 防禦優先：Temp 主要用來「保留防禦牌或回復牌」，避免壞手牌暴斃。
- 節省資源：Temp 主要用來「存牌等配方」，例如缺一張就能組出回資源技能。

## 5. ERA 資料與變數規劃（SAVEDATA 與 TFLAG）
> 命名只是一種建議，重點是：永久保存放 SAVEDATA，戰鬥內暫存放 TFLAG。

### 5.1 戰鬥狀態（TFLAG 為主）
- `TFLAG:BATTLE_TURN`：回合數
- `TFLAG:BATTLE_STRATEGY`：目前策略 ID（0 均衡，1 進攻，2 防禦，3 節省）
- `TFLAG:HAND0..HAND3`：手牌 4 格卡牌 ID（空用 -1）
- `TFLAG:TEMP_CARD`：暫存槽卡牌 ID（空用 -1）
- `TFLAG:DECK_POS`：牌組指標（若你用陣列）
- `TFLAG:DECK_SIZE` / `TFLAG:DISCARD_SIZE`：牌堆計數
- `TFLAG:TEMP_USED_THIS_TURN`：本回合是否已使用 Temp 存取（0 否，1 是）
- `TFLAG:UI_HAND0_SEL` / `TFLAG:UI_HAND1_SEL` / `TFLAG:UI_HAND2_SEL` / `TFLAG:UI_HAND3_SEL`：H0~H3 的 UI 勾選狀態（0 未選取，1 已選取）
- `TFLAG:UI_TEMP_SEL`：Temp 槽位的 UI 勾選狀態（0 未選取，1 已選取）

### 5.2 牌組配置（SAVEDATA）
- `FLAG:DECK_PRESET_ID`：使用哪套牌組預設
- `FLAG:DECK_CARD_POOL[n]`：牌池卡牌 ID（若你用固定表）
- `FLAG:DECK_CARD_COUNT[n]`：對應卡牌數量
- `FLAG:UNLOCK_CARD[n]`：卡牌解鎖狀態（工坊製作或事件解鎖）

> 這裡可以與基地系統串起來：工坊製作解鎖新卡，訓練室提升命中或傷害係數，休息室 Buff 影響抽牌前加成。

### 5.3 戰鬥回傳（給日間或晚間結算用）
- `RESULT:WINLOSE`：勝敗
- `TFLAG:BATTLE_TURNS`：總回合數
- `TFLAG:BATTLE_LOOT_*`：掉落與資源增減
- `TFLAG:BATTLE_INJURY_*`：傷勢或狀態異常

## 7. 風險與對策
- **映射表越做越亂**：把配方做成 CSV 表，戰鬥內只讀表與比對，別把配方寫死在流程裡。
- **Temp 變成強制最佳解**：堅持每回合只允許一次存取，且存牌後不補抽，才有成本。
- **AI 選擇看起來很蠢**：先做評分器，再調權重，避免寫死 if else 變成無限補丁。
- **普攻太爛導致拖台錢**：讓普攻低但穩，或加入連續普攻的隱性補正，避免無限磨血。

## 8. 追蹤指標
- 技能觸發率：每場戰鬥技能回合占比
- Temp 使用率：每回合是否使用 Temp，常見存取的牌類型
- 壞手牌頻率：抽到無法組合的比例
- AI 策略切換次數：是否過度抖動
- 戰鬥回合數分布：是否有明顯過長的尾巴
