# Base_Management.md
> 系統：基地經營（設施、房間槽、住宿 NPC）  
> 適用流程：日間階段主操作，晨間產出與晚間清算（由主循環呼叫）

基地經營的核心是三件事：設施提供「可見的操作選項」，房間槽提供「住宿容量與維護成本」，住宿 NPC 提供「成長、加成與事件」。日間讓玩家做選擇與投入資源，晨間把設施與住宿帶來的被動效果一次結算，晚間做疲勞與狀態清算並觸發事件。
把基地系統當成一個可插拔模組，主循環只需要在晨間與晚間各呼叫一次 Tick，在日間提供入口選單即可。

---

## 落地實作清單（先做這 7 件就能跑起來）
- **建立基地入口選單**：在日間提供 `@BASE_MENU`，將訓練室、工坊、休息室、房間管理做成子選單。
- **定義三設施共用規格**：統一「等級、每日使用次數、消耗、效果」欄位，避免每個設施各寫一套。
- **設計房間槽映射**：用「槽位 → NPC 編號」的資料結構，並提供入住、撤離的單一入口函式。
- **拆分晨間與晚間 Tick**：晨間只管產出與維護費，晚間只管疲勞與事件觸發，方便 debug。
- **把 NPC 養成拆成兩條路**：訓練室走「數值成長」，工坊走「裝備與資源能力」，休息室走「好感與狀態增益」。
- **加入每日防呆旗標**：例如 `TFLAG:BASE_DAY_USED_*`、`TFLAG:MORNING_DONE`，避免重複結算與刷資源。
- **規範跳轉只用 CALL/RETURN**：基地內可用 GOTO 轉畫面，離開基地或進戰鬥一律用 CALL/RETURN，回來不迷路。

---

## 系統邊界與呼叫點
### 主循環需要呼叫的入口
- `CALL @BASE_MORNING_TICK`：晨間階段，做設施產出、住宿維護費與加成刷新
- `CALL @BASE_MENU`：日間階段，玩家操作基地
- `CALL @BASE_NIGHT_TICK`：晚間階段，做疲勞與狀態清算、事件池擲骰

> 戰鬥入口由基地選單呼叫：`CALL @BATTLE_ENTER`，戰鬥結束後 `RETURN` 回基地選單原位置。

## 設施系統（Facilities）
### 共通規格
每個設施都遵循下列欄位與流程：

- 等級（LV）：決定可用指令、效率、每日次數上限
- 每日使用次數（Daily Uses）：日內上限，晚間或就寢時清空
- 消耗（Cost）：金錢、材料、補給、時間點數、NPC 疲勞等
- 效果（Effect）：立即生效或延遲到晨間結算
- 解鎖（Unlock）：等級或事件解鎖新選項

### 1) 訓練室（Training Room）
**定位**：用資源與時間換取 NPC 特定素質的穩定成長，屬於可控的養成主渠道。

#### 訓練指令類型（範例，可自行改名）
- 基礎訓練：低成本，少量成長，適合每天做
- 專項訓練：中成本，針對單一素質成長更高
- 強化訓練：高成本，附帶疲勞與受傷風險，成長上限較高

#### 現行訓練流程（實作版）
1. **先選訓練強度**：
   - 基礎訓練：消耗補給 5、疲勞 +20、訓練倍率加成採基礎值。
   - 專項訓練（訓練室 Lv.3+）：消耗補給 10、疲勞 +30、訓練倍率加成採較高值。
2. **選擇目標 NPC**：僅可選住宿 NPC，且疲勞過高者不可執行。
3. **扣除資源後進入側重子選單**：玩家選擇本次訓練主軸（物理 / 魔法 / 特攻 / 物防 / 魔防 / 速度）。
4. **結算具體 BASE 成長**：
   - 基礎訓練：所選 BASE +1
   - 專項訓練：所選 BASE +2
5. **維持原有狀態變動**：同步增加 `CF_FATIGUE` 與 `CF_TRAIN_MOD`，最後輸出包含屬性提升的完成訊息。

#### 推薦成長模型（可調參）
- 成長量 = `基礎成長 + 訓練室等級加成 + 住宿加成 + 狀態修正`  
- 疲勞增加 = `固定值 + 訓練類型係數`  
- 每日次數上限 = `1 + FLOOR(訓練室LV / 2)`（最低 1 次）

#### 可培養的素質範例（對應你專案的 ABL/EXP/能力欄位）
- 體能系：力量、耐力、敏捷
- 技能系：戰術、專精、工作效率
- 社交系：魅力、話術、信任

> 實作時把「素質枚舉」與「成長公式」封裝成 `@SYS_TRAIN_APPLY`，訓練室只負責 UI 與扣款。

---

### 2) 工坊（Workshop）
**定位**：基地的建設與資源轉換中心，提供「擴張」與「長線累積」。

#### 主要功能
- 建設與升級：新增設施、提升設施等級
- 製作與轉換：把材料變成補給、裝備、禮物、消耗品
- 產線管理（建議簡化版）：同一時間只跑 1 條產線，升級後開多格佇列

#### 推薦產出模型（晨間結算）
- 工坊被動產出（若你希望有）：  
  產出 = `基礎產出 + 工坊LV * 係數 + 住宿NPC加成`
- 製作佇列（若你採用）：  
  佇列每晨間前進一次，完成後入庫
- 取消生產（Cancel Production）：玩家可於生產期間隨時中斷生產佇列，中斷後將全額退還 (100%) 該項目的材料成本，讓資源調度具備容錯空間。

#### 工坊如何間接養成 NPC
- 裝備系：製作訓練用具或裝備，提供「成長加成」或「戰鬥加成」
- 資源系：提升補給與材料穩定度，讓訓練室與休息室的每日操作不會斷糧
- 住宿系：建造宿舍擴張，增加房間槽數量，間接提高可培養 NPC 數量

---

### 3) 休息室（Lounge）
**定位**：用互動換取關係與狀態管理，讓養成有情緒曲線與事件出口。

#### 主要功能
- 互動指令：交談、安撫、娛樂、送禮、安排生活
- 狀態回復：降低疲勞、改善心情，提供隔日加成
- 事件觸發：好感達標後解鎖新互動或特殊事件

#### 推薦互動模型
- 每日可互動次數 = `1 + FLOOR(休息室LV / 2)`
- 好感變動 = `基礎值 + 休息室LV加成 + 禮物加成 + NPC 狀態修正`
- 狀態增益（Buff）建議做成「到下一次晨間有效」或「到就寢有效」

#### 休息室如何支援數值養成
- 好感越高，訓練效率加成越高（例如成長量 +5% 到 +20%）
- 心情良好時，疲勞增加減少或晚間事件偏正面

## 房間槽（Room Slot）機制
### 定義
- 基地擁有固定數量的房間槽 `ROOM_CAP`
- 每個住宿 NPC 佔用 1 個槽位
- 槽位是一種「容量限制」與「維護成本」的承諾，等於你把 NPC 納入基地日常運作

### 入住（Move-in）條件
**必備條件**
- 有空槽：`ROOM_USED < ROOM_CAP`
- NPC 可用狀態：不在關卡、未撤退中、未被事件鎖定（由你專案定義）
- 通過入住門檻：例如好感達標、契約完成、支付安置費

**建議條件（做出經營壓力）**
- 基地資源達到最低線：補給大於某數值才允許新增住宿
- 存在對應設施：例如有宿舍或休息室等級達標才可入住

### 撤離（Move-out）條件
**主動撤離**
- 玩家選擇撤離，立刻釋放槽位
- 可設定冷卻：撤離後數日內不可再次入住，避免反覆刷效果

**被動撤離**
- NPC 狀態崩潰：重傷、叛離、事件帶走
- 關係過低：好感跌破下限，觸發離開事件

### 對基地資源的影響（推薦做成可見的帳單）
- 每日維護費（晨間扣除）：`維護費 = 基礎值 + NPC 等級係數`
- 住宿加成（晨間刷新）：不同 NPC 提供不同被動加成
  - 工坊型 NPC：材料產出或製作速度提升
  - 訓練型 NPC：訓練成長加成或疲勞減免
  - 社交型 NPC：休息室互動加成或事件權重調整

> 把「維護費」與「住宿加成」放在晨報中顯示，使玩家更願意擴張房間槽，因為能看得懂錢花去哪。

## 玩家如何用設施培養 NPC（以「可操作」描述）
### 養成路徑 A：訓練室直升數值
1. 日間進入訓練室，先選訓練強度（基礎 / 專項）。
2. 選擇住宿 NPC，確認可訓練狀態後扣除補給。
3. 選擇側重屬性（物理/魔法/特攻/物防/魔防/速度）。
4. 即時提升目標屬性對應 BASE（+1 或 +2）。
5. 同步套用疲勞與訓練倍率變化，晚間再用休息室做狀態管理。

### 養成路徑 B：工坊做裝備與基地能力，拉長線
1. 日間在工坊建造或升級，提高產線與配方解鎖
2. 製作訓練道具或裝備，給 NPC 裝上
3. 裝備提供長期加成，例如訓練成長加成、戰鬥牌組加成、疲勞減免
4. 透過宿舍擴建增加槽位，讓更多 NPC 進入日常循環

### 養成路徑 C：休息室養關係，解鎖指令與事件
1. 日間或晚間進入休息室互動
2. 提升好感與信任，解鎖更高級的訓練或工坊配方
3. 維持心情與疲勞在健康區間，提高次日訓練效率
4. 晚間觸發事件，拿到獨特能力或稀有資源

## SAVEDATA 數據結構規劃（全域變數）
以下命名是「建議規格」，依檔案命名規範調整。重點是區分：
- 永久保存：`FLAG/VAR/CFLAG` 之類的 SAVEDATA
- 當日暫存：`TFLAG` 或每天清空的計數器

### 變數表（小表格）
| 類別 | 建議變數 | 型態 | 說明 | 重置時機 |
|---|---|---|---|---|
| 基地 | `FLAG:BASE_DAY` | SAVEDATA | 經營天數或日期 | 不重置 |
| 資源 | `VAR:BASE_RES_MONEY` / `VAR:BASE_RES_MAT` / `VAR:BASE_RES_SUP` | SAVEDATA | 金錢、材料、補給等 | 不重置 |
| 設施等級 | `FLAG:BASE_FAC_LV_TRAIN` / `FLAG:BASE_FAC_LV_WORK` / `FLAG:BASE_FAC_LV_LOUNGE` | SAVEDATA | 三設施等級 | 不重置 |
| 設施使用次數 | `TFLAG:BASE_USES_TRAIN` / `TFLAG:BASE_USES_LOUNGE` | TFLAG | 今日已使用次數 | 就寢或晚間 Tick |
| 工坊佇列 | `FLAG:WORK_Q_TYPE[n]` / `FLAG:WORK_Q_REMAIN[n]` | SAVEDATA | 產線類型與剩餘天數 | 不重置 |
| 房間容量 | `FLAG:ROOM_CAP` / `FLAG:ROOM_USED` | SAVEDATA | 槽位上限與已用數 | 不重置 |
| 槽位歸屬 | `FLAG:ROOM_OWNER[slot]` | SAVEDATA | 槽位對應 NPC 編號，空槽用 -1 | 不重置 |
| NPC 住宿狀態 | `CFLAG:CHARA:IS_RESIDENT` / `CFLAG:CHARA:ROOM_SLOT` | SAVEDATA | 是否住宿與槽位編號 | 不重置 |
| NPC 成長加成 | `CFLAG:CHARA:TRAIN_MOD` / `CFLAG:CHARA:MOOD` / `CFLAG:CHARA:FATIGUE` | SAVEDATA | 訓練倍率、心情、疲勞 | 不重置 |

## 指令跳轉與選單路由（建議 Label）
### 日間基地入口
- `@PHASE_DAY` 內：玩家選「基地」后 `CALL @BASE_MENU`

### 基地總選單
- `@BASE_MENU`
  - 訓練室：`GOTO @BASE_TRAIN_MENU`
  - 工坊：`GOTO @BASE_WORK_MENU`
  - 休息室：`GOTO @BASE_LOUNGE_MENU`
  - 房間管理：`GOTO @BASE_ROOM_MENU`
  - 出戰：`CALL @BATTLE_ENTER`，結束後 `RETURN` 回此選單
  - 離開基地：`RETURN`

## 風險與對策
- **刷資源與重複結算**：晨間 Tick 設 `TFLAG:MORNING_DONE`，同一天只允許結算一次。
- **房間槽狀態不同步**：入住與撤離只能走 `@SYS_ROOM_MOVE_IN / @SYS_ROOM_MOVE_OUT`，禁止手動改變 `ROOM_USED`。
- **訓練變成唯一最優解**：用每日次數上限、疲勞成本、工坊長線收益把策略拉開。
- **休息室變成純加好感按鈕**：加入事件權重、狀態管理與互動次數限制，讓它是經營工具而不是點擊器。

## 追蹤指標
- 每日晨間：維護費總額、產出總額、住宿加成總額
- 日間：三設施使用率，出戰前平均操作步數
- NPC：平均成長速度、疲勞與心情分布、撤離觸發原因
- 房間槽：入住率、平均空槽天數、擴建後回本天數
