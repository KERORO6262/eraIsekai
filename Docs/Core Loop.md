# Core_Loop.md
> 專案：ERA（Emuera 相容）養成 + 策略經營 + 戰鬥闖關  
> 主題：每日流程（Day Cycle）與指令跳轉邏輯（CALL/RETURN 為主）

本文件定義「一天」的主循環：晨間負責基地產出與資源結算，日間與晚間提供玩家操作與互動，並可隨時進入「戰鬥階段」完成關卡後返回原階段。整體採用 ERA 常見的「中央選單 + 階段子選單」結構，並以 `CALL/RETURN` 維持返回點，避免迷路式跳轉。
每日流程當成一個狀態機，主程式只做分派，細節交給各階段模組。戰鬥被視為可插拔的「額外階段」，不改寫主循環，只回傳結果與消耗。

---

## 執行
- **建立單一入口**：用 `@MAIN` 或 `@GAME_LOOP` 統一進入每日循環，避免多入口造成狀態不同步。
- **用階段旗標控流程**：用 `FLAG:DAY_PHASE` 或同等變數表示晨間、日間、晚間，主迴圈只做 `SWITCH` 分派。
- **用 CALL/RETURN 保障回來**：進戰鬥一律 `CALL @BATTLE_ENTER`，結束後 `RETURN` 回呼叫點，少用跨檔 `GOTO`。
- **把資源結算集中化**：晨間產出、晚間清算都走同一套「加總器」與「日結報表」，方便日後加新資源。
- **把 NPC 互動做成設施指令的一種**：在日間的基地選單中，互動與設施操作共用選單框架，只是路由到不同 handler。
- **把隨機事件設成可回放記錄**：晚間事件觸發後寫入 `TFLAG` 或 log，方便 debug「今天為何爆炸」。
- **戰鬥回傳要結構化**：至少回傳「勝敗、耗時/回合、資源變動、狀態異常」，讓日間與晚間能做後續演出與結算。
- **加一層防呆**：任何階段若偵測到狀態非法（例如資源負值、房間槽不足），立刻導向 `@SYS_RECOVER` 做修正或提示。

## 每日流程總覽（Day Cycle）
### 核心概念
- **一天由 3 個主階段構成**：晨間 → 日間 → 晚間 →（下一天）
- **戰鬥是額外階段**：可從日間或晚間進入，完成後回到原階段，不直接推進到下一階段
- **推進階段的權限集中**：只有主循環能把 `DAY_PHASE` 往下推進，避免子模組亂改

---

## 流程表（小表格）
| 階段 | 主要內容 | 建議入口 Label | 可進戰鬥 | 離開條件（推進） | 建議使用的狀態 |
|---|---|---|---|---|---|
| 晨間 | 設施產出、資源結算、簡報 | `@PHASE_MORNING` | 否 | 玩家按「開始日間」或自動進入 | `FLAG:DAY`、`FLAG:DAY_PHASE=1` |
| 日間 | 基地操作、設施指令、NPC互動、可進關卡戰鬥 | `@PHASE_DAY` | 是 | 玩家選「結束日間」 | `TFLAG:RETURN_PHASE=2` |
| 晚間 | 數值清算、隨機事件、收尾，亦可進戰鬥 | `@PHASE_NIGHT` | 是 | 玩家選「就寢」結束一天 | `TFLAG:RETURN_PHASE=3` |
| 戰鬥 | 抽牌 4 格、組合映射技能、未命中普攻、暫存槽每回合一次、自動戰鬥策略 | `@BATTLE_ENTER` | 進入即是 | 結束後 `RETURN` | `RESULT:WINLOSE`、`TFLAG:BATTLE_*` |

> `TFLAG:RETURN_PHASE` 用來標記「我從哪個主階段進來的」，若你全程使用 `CALL/RETURN`，它主要用於 UI 顯示與除錯。

## 指令跳轉邏輯（符合 ERA 常見寫法）
### 原則
1. **跨階段使用 CALL/RETURN**：確保戰鬥結束可以穩定回到呼叫點。  
2. **同階段內部可以 GOTO**：用於同檔案的選單跳轉，但避免跨檔亂跳。  
3. **主循環掌管推進**：晨間進日間、日間進晚間、晚間進隔日，一律由主循環統一處理。  

### 建議的主循環骨架（概念稿，非強制語法）
- `@GAME_LOOP`
  - 初始化新的一天（若需要）
  - `FLAG:DAY_PHASE = 1`（晨間）
  - `WHILE 1`
    - `IF FLAG:DAY_PHASE == 1` → `CALL @PHASE_MORNING`
    - `ELSEIF FLAG:DAY_PHASE == 2` → `CALL @PHASE_DAY`
    - `ELSEIF FLAG:DAY_PHASE == 3` → `CALL @PHASE_NIGHT`
    - `ELSE` → break（或修正）
  - 進入下一天

> 重點是：每個 `@PHASE_*` 結束時只回傳「玩家選了什麼」，由主循環決定是否推進 `DAY_PHASE`。

## 晨間階段（Morning Phase）
### 目標
- 把基地「被動產出」一次結算清楚
- 給玩家一份簡短可讀的晨報，告訴他今天能做什麼

### 建議流程
1. **設施產出結算**
   - 對每個設施做：產出量 = 基礎值 + 升級加成 + NPC/房間加成 + 異常修正
   - 寫入資源池（例如：金錢、材料、補給、情緒/士氣類）
2. **維護與消耗結算**
   - 住房維護費、補給消耗、狀態衰減（例如疲勞回復或自然下降）
3. **晨間簡報 UI**
   - 顯示：昨日總結（可選）、今日新增資源、基地狀態警示
4. **推進到日間**
   - 玩家按「開始日間」後，回傳主循環 → `FLAG:DAY_PHASE = 2`

### 常見踩坑與對策
- **產出被重複結算**：用 `TFLAG:MORNING_DONE` 之類的當日旗標防重入。
- **資源出現負值**：晨間結算前後都做一次 clamp 或警示，並記錄 log。

---

## 日間階段（Day Phase）
### 目標
- 讓玩家在基地內做「主操作」：建設、升級、派工、製作、訓練、互動
- 提供「進入關卡戰鬥」的入口，打完回來繼續逛基地

### 建議的日間選單結構
- 基地管理
  - 建設/升級設施
  - 分配人手或運行策略
  - 製作/研究
- 住宿 NPC 互動（養成）
  - 交談、訓練、送禮、安排房間
  - 互動結果影響：好感、能力成長、可解鎖指令
- 外出與戰鬥
  - 選關卡 → `CALL @BATTLE_ENTER`
  - 戰鬥結束 → `RETURN` 回日間選單
- 結束日間
  - 回傳主循環 → `FLAG:DAY_PHASE = 3`

### 戰鬥返回點的建議做法
- 進戰鬥前：記錄 `TFLAG:RETURN_PHASE = 2`（日間）
- 戰鬥結束後：只寫入結果與資源變動，不要直接推進日夜
- 回到日間後：用「戰鬥結果摘要」提示玩家，並把戰利品加入背包

---

## 晚間階段（Night Phase）
### 目標
- 做日末清算，觸發特殊隨機事件
- 提供晚間專屬指令（例如巡邏、夜訓、休息配置），並允許進戰鬥

### 建議流程
1. **日末數值結算**
   - 疲勞、飢餓、心情、關係值的自然變動
   - 當日任務與目標的結算（完成度、獎勵、懲罰）
2. **隨機事件觸發（Special Random Events）**
   - 事件池：基地事件、NPC事件、外部勢力事件、資源事件
   - 條件判斷：設施等級、資源門檻、NPC 狀態、旗標進度
   - 事件結果：提供抉擇，並把結果寫入 log
3. **晚間可選行動**
   - 晚間互動、夜訓、安排隔日策略
   - 可選進戰鬥：`CALL @BATTLE_ENTER`，結束後回晚間
4. **就寢收尾**
   - 玩家選「就寢」後，回傳主循環 → 推進到下一天（`FLAG:DAY += 1`）

### 隨機事件的實務建議
- 用「權重 + 冷卻」控制重複：避免同一事件連續三天來敲門。
- 每個事件要有「可回滾的最小記錄」：至少記錄事件 ID、結果分支、資源變動。

---

## 額外階段：戰鬥階段（Battle Phase）
### 目標
- 執行關卡內的抽牌戰鬥流程
- 把結果回傳給呼叫端（日間或晚間），並讓呼叫端做後續演出與結算

### 進入與退出規則
- **進入**：日間或晚間選單中選擇關卡 → `CALL @BATTLE_ENTER`
- **退出**：戰鬥流程結束後只做：
  - 寫入結果（勝敗、回合、掉落、狀態）
  - `RETURN` 回到原本階段，不直接改 `DAY_PHASE`

### 建議回傳資訊（最低需求）
- `RESULT:WINLOSE`：0 敗北、1 勝利（或你習慣的編碼）
- `TFLAG:BATTLE_TURNS`：回合數
- `TFLAG:BATTLE_LOOT_*`：掉落與資源增減
- `TFLAG:BATTLE_INJURY_*`：傷勢或狀態異常（若有）

---

## 追蹤指標
- 每日晨間結算耗時：是否隨設施數量線性暴增
- 玩家日間操作步數：從進日間到出戰需要幾次按鍵
- 戰鬥返回正確率：從日間進戰鬥回日間、從晚間進戰鬥回晚間是否 100%
- 隨機事件重複率：同一事件在 N 天內出現的次數
- 資源異常次數：負值、溢位、重複結算的偵測與 log 次數
