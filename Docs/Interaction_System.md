# Interaction_System.md
> 系統：互動指令（日常互動 + 成人向互動）  
> 目標：用 PALAM 與角色長期狀態決定指令成功率與效果，並把結果回饋到基地設施效率與住宿狀態  
> 注意：本文件只定義規範與數值流程，不包含任何露骨文本

---

## 1. 核心概念與設計原則
### 1.1 兩條互動線
- **日常互動**：主打好感、心情、壓力管理。高容錯，高一致性，適合每日操作。
- **成人向互動**：主打興奮成長、開發經驗、刻印與解鎖。高變動，高回報，同時也高風險。

### 1.2 一句話總結判定
- 日常互動看「關係與狀態」。
- 成人向互動看「關係、狀態、反感門檻與刺激負荷」。

## 2. 指令分類與資料結構
### 2.1 指令資料欄位（建議做成 Interaction.csv 或 ERB 常數表）
| 欄位 | 說明 |
|---|---|
| CMD_ID | 指令唯一 ID |
| CATEGORY | DAILY / ADULT |
| TAGS | 例如 RELAX、BOND、TRAIN、RISK、VIRGIN_CHECK、AFTERCARE |
| REQUIRE_BOND | 最低好感段位門檻（BOND_LV） |
| MAX_DISGUST | 反感上限門檻，超過會大幅降成功率或直接禁止 |
| COST_FATIGUE | 疲勞消耗（或增加） |
| COST_RESOURCE | 金錢、道具消耗（可選） |
| PALAM_DELTA_BASE | 期望變動：快樂、痛苦、反感、興奮（基礎值） |
| EXP_GAIN | 增加的互動經驗或開發經驗 |
| MARK_POINT | 刻印點數增量規則（快樂刻印、苦痛刻印等） |
| FAIL_RULE | 失敗時 PALAM/狀態變動 |
| HOOK | ERB 鉤子段落名，例：INT_2001 |

### 2.2 指令分類範例（示意）
| 類別 | 子類 | 目的 | 常見 TAG |
|---|---|---|---|
| 日常互動 | 交流、安撫、娛樂、送禮 | 提升好感，降低壓力，調整心情 | BOND、RELAX |
| 成人向互動 | 開發、試探、進階、善後 | 提升興奮與經驗，累積刻印，解鎖新指令 | RISK、VIRGIN_CHECK、AFTERCARE |

## 3. 參照數值與 PALAM.csv 需求
### 3.1 PALAM.csv 最小需求欄位
| PALAM 欄位 | 本文件用途 | 建議範圍 |
|---|---|---|
| 快樂 | 正向接受度與回饋倍率 | 0..100 |
| 痛苦 | 負荷與風險來源，也可成為特定刻印來源 | 0..100 |
| 反感 | 硬門檻，決定拒絕與負面後果 | 0..100 |
| 興奮（或淫欲） | 成人向互動的推進資源與事件門檻 | 0..100 |

> 若 PALAM.csv 已有相近欄位，可以做對映，不需要新增欄。

### 3.2 Data_Structure.md 既有欄位依賴
- `CFLAG:CHARA:BOND_LV`：好感段位，解鎖、成功率、回饋倍率  
- `CFLAG:CHARA:MOOD`：心情，影響日常互動成功率與回饋  
- `CFLAG:CHARA:FATIGUE`：疲勞，影響兩類互動的失敗風險  
- `CFLAG:CHARA:TRAIN_MOD`：訓練倍率，互動可調整以回饋基地  
- `CFLAG:CHARA:IS_RESIDENT / ROOM_SLOT / MOVEOUT_COOLDOWN / MAINT_COST_MOD`：住宿狀態與成本，互動可影響入住與撤離壓力

## 4. 數值判定邏輯
互動指令的判定分兩段：
1) **可執行判定**：避免在高反感或極端疲勞下硬推造成系統崩壞  
2) **成功率與效果倍率**：同一指令在不同狀態下表現不同

### 4.1 可執行判定（Gate）
- 若 `反感 >= MAX_DISGUST` 且指令是 ADULT：  
  - 建議直接禁止或強制走「善後/緩和」類指令
- 若 `疲勞 >= 90`：  
  - 允許 DAILY 的輕量指令  
  - ADULT 指令改成高風險或禁止

### 4.2 成功率（SuccessRate）
#### A. 日常互動
- **AffinityScore（日常）** = `BOND_LV*8 + MOOD*0.4 - FATIGUE*0.3 - 反感*0.6`
- **SuccessRate（日常）** = `CLAMP(10, 95, 60 + AffinityScore)`

#### B. 成人向互動
- **ConsentScore（成人向）** = `BOND_LV*10 + MOOD*0.2 + 興奮*0.4 - 反感*1.2 - FATIGUE*0.4`
- **SuccessRate（成人向）** = `CLAMP(5, 90, 40 + ConsentScore)`

> 把 “60 / 40” 當成基準值，用於早期調參。

### 4.3 效果倍率（EffectScale）
- **EffectScale** = `1.0 + (SuccessRate - 50) / 100`
- 另外加一個「反感懲罰」：  
  - 若 `反感 >= 70`，EffectScale 再乘 `0.6`  
  - 若 `反感 <= 20`，EffectScale 乘 `1.1`

### 4.4 失敗處理（Fail Rule）
失敗不要只有 0 收益，建議採三段：
- **小失敗**：快樂小幅下降，反感小幅上升，給玩家明確訊號
- **大失敗**：反感上升更多，痛苦上升，並可能觸發撤離壓力
- **災難級**：只有在極端狀態或硬門檻下才出現，並且必須有緩和出口（善後、休息、隔離）

## 5. 經驗與刻印
### 5.1 經驗類型（建議）
| 類型 | 建議儲存 | 作用 |
|---|---|---|
| 日常互動經驗 | `CFLAG:CHARA:BOND_LV` 或獨立 EXP | 解鎖日常指令、提升互動效率 |
| 成人向開發經驗 | 新增 `CFLAG:CHARA:ADULT_EXP` | 解鎖進階指令、降低反感增長 |
| 刻印點數 | 新增 `CFLAG:CHARA:MARK_*_PT` | 轉成刻印等級 |

### 5.2 絕頂（事件型回饋）
> 以系統事件描述，不輸出任何文本細節。

**觸發條件建議**
- `興奮 >= 90` 且 `快樂 >= 70`  
- `反感 <= 40`  
- 本次指令 EffectScale >= 1.0

**觸發效果建議**
- `興奮` 大幅下降（回到 20..40）  
- `快樂` 小幅上升  
- 增加 `ADULT_EXP`  
- 增加「快樂刻印點數」

### 5.3 處女喪失（一次性旗標）
> 以 “VIRGIN_CHECK” 的指令標記決定是否檢查。

**變數建議**
- 新增 `CFLAG:CHARA:VIRGIN = 1/0`

**獲取條件建議**
- 首次成功執行帶 `VIRGIN_CHECK` 的成人向指令
- `反感` 需低於某門檻（例如 60），避免不合理的硬推路線

**效果建議**
- 設定 `VIRGIN = 0`
- 給一次性經驗獎勵或事件解鎖
- 同步寫入 log 方便除錯

### 5.4 刻印（Mark）
刻印採「點數累積 → 升級」比較穩，避免一次事件爆炸。

#### A. 苦痛刻印（Pain Mark）
- **點數增加**：  
  - `max(0, Δ痛苦 - Δ快樂) + (反感>=60 ? 2 : 0)`
- **升級門檻**（示例）：  
  - Lv1: 20 點，Lv2: 60 點，Lv3: 120 點
- **系統效果建議**：  
  - 高等級時，某些成人向指令的風險下降或結果分支不同  
  - 同時提高「撤離壓力」的敏感度，避免變成單向最優解

#### B. 快樂刻印（Pleasure Mark）
- **點數增加**：  
  - `max(0, Δ快樂 + Δ興奮 - Δ反感)`
- **升級門檻**（示例）：  
  - Lv1: 20 點，Lv2: 60 點，Lv3: 120 點
- **系統效果建議**：  
  - 提高成人向互動成功率下限  
  - 增加絕頂事件的觸發概率或收益

## 6. 養成回饋到基地經營（Base_Management 連動）
互動結果需要直接影響你基地的三件事：訓練效率、休息室收益、住宿穩定度。

### 6.1 回饋到設施效率
- **訓練室效率**：  
  - 以 `TRAIN_MOD` 作為最終倍率入口  
  - 規則建議：  
    - 快樂刻印或高好感，提高 `TRAIN_MOD`（例如 +1..+3）  
    - 高反感或高疲勞，降低 `TRAIN_MOD`（例如 -1..-3）
- **休息室互動收益**：  
  - 若 `MOOD` 高，日常互動的效果倍率加成  
  - 若 `FATIGUE` 高，休息室的回復量提高，但日常互動增益降低

> 這段會對齊基地文件的設定：休息室提升好感與心情，並可提高訓練效率。

### 6.2 回饋到住宿狀態（入住與撤離壓力）
- **入住門檻調整**  
  - 若 `BOND_LV` 達標且 `反感` 低，入住成功率提高  
- **撤離壓力**  
  - 若 `反感` 長期高於門檻，或連續大失敗：  
    - 增加撤離機率，並設定 `MOVEOUT_COOLDOWN`  
  - 若互動以善後類指令穩定關係：  
    - 降低撤離壓力  
- **維護費修正**  
  - 可用 `MAINT_COST_MOD` 表達住宿合作度或需求提升  
  - 只做小幅波動，避免經濟系統被互動單點刷穿

## 7. 執行流程（ERB 實作順序建議）
1. 讀取指令資料：CMD_ID → CATEGORY/TAGS/門檻/變動
2. Gate：檢查反感、疲勞、入住狀態、冷卻
3. 計算 SuccessRate → 擲骰
4. 計算 EffectScale → 套用 PALAM 變動與 EXP
5. 結算刻印點數 → 若達門檻升級
6. 回饋到 `BOND_LV / MOOD / FATIGUE / TRAIN_MOD`
7. 檢查撤離壓力或事件觸發
8. 寫 log（推薦）：分數、倍率、ΔPALAM、刻印點數

---

## 9. 追蹤指標（調參用）
- 指令使用分布：DAILY vs ADULT 佔比
- 反感的平均值與尾端：是否常常衝到硬門檻
- 大失敗率：是否造成撤離過於頻繁
- TRAIN_MOD 的平均變動：是否被互動推到失控
- 刻印升級速度：到 Lv1、Lv2、Lv3 的平均天數
