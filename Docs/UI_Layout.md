# UI_Layout.md
> 目的：定義 ERA（Emuera）文字介面的佈局與視覺層次，讓「基地經營」與「抽牌戰鬥」在同一套 UI 語言下運作  
> 關鍵約束：採用「中央選單 + 子選單」的指令跳轉，戰鬥以 `CALL @BATTLE_ENTER` 進入並 `RETURN` 回原階段。

本文件把畫面拆成三層：頂部固定資訊（資源與狀態 Bar）、中段內容區（基地或戰鬥內容）、底部操作區（按鈕列表）。
基地主畫面要能在單一螢幕同時看見資源、設施等級與住宿 NPC，讓玩家不需要進選單才知道今天能做什麼。戰鬥介面則以 4 個手牌槽與 1 個暫存槽為核心，並在同畫面提供技能映射提示與策略切換。整體按鈕 ID 採區間管理，避免系統選項與角色操作互相撞號，後續擴充也好控。

---

## 先做就能一致的 6 條 UI 規範
- **固定頂欄**：永遠顯示 Day Phase、資源摘要與 1 至 3 條關鍵 Bar，玩家不用翻頁找狀態。
- **統一三區塊**：頂欄（固定）+ 內容區（可捲動）+ 操作區（固定），每個畫面都長一樣，降低學習成本。
- **基地單屏總覽**：基地畫面一屏顯示資源、設施等級、住宿 NPC 清單，對齊基地選單入口 `@BASE_MENU`。
- **戰鬥操作前置**：抽牌後先顯示 Temp 與 4 格手牌，提供「存牌 / 取牌」兩按鈕，且本回合鎖定一次操作。
- **映射提示即時**：玩家選牌時即時列出可觸發技能與預估效果，沒組合就提示會普攻，避免無回饋。
- **按鈕區間分流**：0-99 系統，100+ 角色與內容操作，所有模組共用同一套表格，不靠記憶硬背。

| 區域 | 位置 | 內容 | 更新頻率 |
|---|---|---|---|
| Top HUD | 螢幕最上方 2 至 4 行 | Day/Phase、資源、Bar | 每次指令後 |
| Main Panel | 中段 | 基地總覽或戰鬥 log 與卡槽 | 每次畫面刷新 |
| Command Panel | 最下方 | 按鈕列表、頁面切換、返回 | 每次畫面刷新 |

## 1. 通用 UI 元件（全畫面共用）
### 1.1 Top HUD（固定資訊）
固定包含：
- Day 與 Phase（晨間/日間/晚間/戰鬥）
- 基地核心資源（例：金錢、材料、補給）
- 3 條關鍵 Bar（HP、氣力、壓力或心情）

> 互動系統會吃 PALAM 與角色長期狀態做判定，因此「壓力 / 心情 / 疲勞」類 Bar 放在固定區最有價值。

### 1.2 Main Panel（內容區）
- 基地：總覽資訊 + 住宿 NPC 列表 + 今日提示
- 戰鬥：戰鬥 log、敵我狀態摘要、技能提示區、卡槽區

### 1.3 Command Panel（操作區）
- 一律採「短行按鈕」列表，並保留 3 個系統鍵：
  - 返回
  - 上一頁/下一頁（若有分頁）
  - 說明

## 2. 主畫面佈局（基地總覽）
### 2.1 顯示目標
同一螢幕內要看到：
1) 基地資源  
2) 當前設施等級（訓練室、工坊、休息室）  
3) 正在住宿的 NPC 列表（槽位、狀態摘要）

基地系統核心在「設施、房間槽、住宿 NPC」三件事，因此 UI 也應圍繞三件事排版。

### 2.2 資源與設施等級（Top HUD 內或 Main Panel 第一段）
- 資源：顯示 3 到 5 個最常用資源，其他資源放詳細頁
- 設施等級：以一行顯示
  - 訓練室 LV
  - 工坊 LV
  - 休息室 LV

### 2.3 住宿 NPC 列表（Main Panel 主體）
- 每列建議包含：
  - Slot 編號
  - NPC 名稱
  - 狀態三要素：好感段位、心情、疲勞（或你想顯示的三個欄位）
- 點選 NPC 進入 NPC 子選單（互動、訓練、房間管理）

> NPC 的好感與狀態欄位在資料結構中已明確存在，例如 `BOND_LV / MOOD / FATIGUE / TRAIN_MOD`。

### 2.4 基地操作入口（Command Panel）
基地選單路由建議固定如下，避免玩家迷路：`@BASE_MENU` 內含訓練室、工坊、休息室、房間管理、出戰與離開。

#### 單屏基地總覽示意（文字版）
- Top HUD  
  - Day 12 | 日間 | 金錢 1200 | 材料 35 | 補給 18  
  - 訓練 LV2 | 工坊 LV1 | 休息 LV3 | 房間 6/8  
  - HP Bar | 氣力 Bar | 壓力 Bar
- Main Panel  
  - 今日提示：工坊佇列剩 1 天，晚間可能觸發事件  
  - 住宿 NPC  
    - [S0] NPC_A | 好感 3 | 心情 70 | 疲勞 20  
    - [S1] NPC_B | 好感 1 | 心情 45 | 疲勞 60  
- Command Panel  
  - [100] 訓練室  
  - [110] 工坊  
  - [120] 休息室  
  - [130] 房間管理  
  - [140] 出戰  
  - [0] 返回

## 3. 戰鬥介面設計
戰鬥以「固定 4 格手牌」與「固定 1 格暫存槽」為每回合操作面，抽牌後必須顯示 Temp 與手牌並提供存取選項，完成後才進入選牌與技能映射。

### 3.1 戰鬥畫面分區
- Top HUD：HP、能量、策略、回合數
- Main Panel 上半：戰鬥 log（最近 6 到 10 行）
- Main Panel 下半：技能映射提示區
- Bottom 卡槽區：Hand Slots 4 格 + Temp Slot 1 格
- Command Panel：選牌、存取、確認、策略切換、自動戰鬥

### 3.2 卡槽位置與顯示格式
#### Hand Slots（4 格）
- 建議排成一行，從左到右固定位置：
  - [H0] [H1] [H2] [H3]
- 每格顯示：
  - 卡名
  - 子類別（A/B/D/S/P）
  - 能量消耗（若有）
  - 選取狀態（例如加上 `*`）

#### Temp Slot（1 格）
- 放在 Hand 行的右側或下一行左側，視寬度而定：
  - [T] Temp: 卡名 或 Empty
- 額外顯示本回合是否已使用 Temp（例如 `TempUsed: 0/1`），因為每回合只允許一次存取。

### 3.3 技能映射提示區（Dynamic Hint）
玩家選牌時，提示區即時刷新：
- 已選牌集合（例：A + B）
- 候選技能列表（最多顯示 3 個）
  - 技能名
  - 需求牌
  - 預估效果（預估傷害、護盾、狀態）
  - 能量消耗
- 若無技能可觸發：
  - 顯示「本回合將執行普攻」提示（保底不會卡住）

### 3.4 策略與自動戰鬥顯示
- 顯示目前策略：均衡 / 進攻 / 防禦 / 節省
- 若啟用自動戰鬥：
  - 顯示 Auto: ON
  - 顯示策略是否鎖定
- 策略切換的行為符合戰鬥模組設計：策略是權重不同的評分器，不改變底層規則。

#### 戰鬥畫面示意（文字版）
- Top HUD  
  - Turn 3 | Strategy: 防禦 | Auto: OFF  
  - HP 78/100 | EN 4/10 | TempUsed 0
- Battle Log  
  - 你使用 格擋架勢，獲得護盾  
  - 敵人攻擊，被減傷  
- Skill Hint  
  - 已選：D + D  
  - 候選：格擋架勢（護盾 +X）  
- Card Slots  
  - [H0] 格擋(D)  [H1] 斬擊(A)  [H2] 格擋(D)  [H3] 專注(P)   [T] Empty
- Command Panel  
  - [410] 選 H0  [411] 選 H1  [412] 選 H2  [413] 選 H3  
  - [420] Temp 存牌  [421] Temp 取牌  
  - [430] 確認出招  
  - [440] 切換策略  
  - [450] 自動戰鬥 ON/OFF  
  - [0] 返回

## 4. 自訂 Bar 條（BAR 指令規範）
### 4.1 Bar 位置建議
- 常駐在 Top HUD 的最後 1 到 2 行
- 每條 Bar 固定寬度與固定標籤，避免畫面抖動

### 4.2 Bar 命名與資料來源
戰鬥用 HP 與能量可直接對齊 BASE.csv 的欄位，例如 `BTL_HP / BTL_MAXHP / BTL_EN / BTL_MAXEN`。  
Temp 與策略也可用 BASE 表達（`BTL_TEMP_CARD / BTL_TEMP_USED / BTL_STRATEGY`），或留在 TFLAG。

### 4.3 BAR 指令樣式（示例）
> 下面是格式示意，實際 BAR 指令參數依你使用的 Emuera 版本調整。

~~~erb
; Top HUD Bars（固定寬度 20）
BAR "HP"    BASE:BTL_HP     BASE:BTL_MAXHP   20
BAR "EN"    BASE:BTL_EN     BASE:BTL_MAXEN   20

; 基地常駐（示例：壓力 / 心情 / 疲勞 三選一或輪替）
BAR "Mood"  CFLAG:TARGET:MOOD     100         20
BAR "Fat"   CFLAG:TARGET:FATIGUE  100         20
~~~

> 互動系統會以「心情、疲勞、反感」影響成功率與效果，所以把 Mood 與 Fatigue 做成常駐 Bar，玩家會更懂為何指令成功率在飄。

## 5. 按鈕索引（Button ID）區間規範
### 5.1 核心原則
- **0-99 為系統**：返回、翻頁、說明、確認、取消、模式切換
- **100 以上為內容操作**：基地、角色、互動、戰鬥等
- 每個畫面至少保留一顆系統返回鍵（ID 0）

### 5.2 建議區間表
| 區間 | 類別 | 內容 |
|---:|---|---|
| 0-9 | 系統導覽 | 返回、上一頁、下一頁、說明 |
| 10-49 | 系統模式 | 自動戰鬥 ON/OFF、策略切換、顯示切換 |
| 50-99 | 系統保留 | Debug、測試、開發期指令 |
| 100-159 | 基地主選單 | 訓練室、工坊、休息室、房間管理、出戰  |
| 160-199 | 基地子操作 | 建設、升級、製作、佇列管理 |
| 200-299 | NPC 選擇與管理 | 選 NPC、入住、撤離、房間調整 |
| 300-399 | 互動指令 | 日常互動、成人向互動、善後，依類別分頁  |
| 400-459 | 戰鬥選牌 | 選 H0..H3、清除選取、確認出招 |
| 460-499 | Temp 與策略 | 存牌、取牌、自動、策略鎖定，因 Temp 每回合一次需特別顯示 |

### 5.3 避免撞號的實務技巧
- 每個子選單先定義「起始偏移量」，例如基地子選單固定從 160 開始。
- 一個畫面最多顯示 12 到 16 個按鈕，超過就分頁。
- 「NPC 列表」採 200-299 區間時，索引可用 `200 + idx` 映射，idx 由清單排序決定。
